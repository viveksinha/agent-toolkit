# execution-skills

**Execution skills — meeting processing, stakeholder comms, support triage, and idea stress-testing.**

## Skills

| Skill | What it does |
|-------|-------------|
| [meeting-synthesis](skills/meeting-synthesis/SKILL.md) | Turn meeting transcripts or notes into concise takeaways and action items with DRIs |
| [stakeholder-update](skills/stakeholder-update/SKILL.md) | Write a weekly stakeholder update — metrics, bets, customer signal, highlights |
| [yes-and](skills/yes-and/SKILL.md) | Build on an idea constructively — find what's working, extend it, surface risks with mitigations |
| [pm-weekly-brief](skills/pm-weekly-brief/SKILL.md) | Read Slack, calendar, and Drive activity from the past 7 days and write a personal PM summary |
| [release-notes-generator](skills/release-notes-generator/SKILL.md) | Read completed tickets and write polished, user-facing release notes grouped by product area |
| [update-knowledge-base](skills/update-knowledge-base/SKILL.md) | Cross-reference shipped tickets with your help center to detect outdated or missing docs |
| [kinder-messages](skills/kinder-messages/SKILL.md) | Rephrase Slack/email/DMs to be warmer and collaborative without losing clarity |
| [support-feedback-prioritization](skills/support-feedback-prioritization/SKILL.md) | Pull Intercom + Slack signals, enrich with CRM data, and rank by customer value into a tiered report |

## Commands

| Command | What it does |
|---------|-------------|
| [yc-office-hours](commands/yc-office-hours.md) | Interactive stress-test of a product idea with branching modes and forcing questions before committing to build |
